## 🌳 Branching Strategy
This project follows a **feature-based branching strategy**.

### 1. Main Branch: `main`
* **Status**: Always stable and runnable.
* **Purpose**: Used for demo and integration.
* **Rule**: Avoid direct commits unless absolutely necessary.

### 2. Feature Branch: `feature/<feature-name>`
* **Purpose**: Used for implementing new features.
* **Examples**:
    * `feature/homepage-ui`
    * `feature/service-list-ui`
    * `feature/booking-flow-ui`
    * `feature/customer-dashboard-ui`
    * `feature/specialist-treatment-plan-ui`
    * `feature/admin-service-catalog-ui`

### 3. Fix Branch: `fix/<bug-name>`
* **Purpose**: Used for bug fixes.
* **Examples**:
    * `fix/login-error`
    * `fix/booking-validation`
